Encryption Architecture

How CortexGuard keeps your credentials safe across all cloud providers

🔒 What does CortexGuard encrypt and why?
Your cloud credentials (AWS keys, Azure secrets, GitHub tokens) are sensitive. If leaked, attackers could access your entire cloud infrastructure. CortexGuard generates a unique encryption key for your session, encrypts all credentials before use, and never stores them on disk. The private key lives only in your browser's memory and is deleted when you close the tab.

▶ How Agents Communicate & Reason
The Orchestrator LLM (GPT-4o / Claude) manages the agent lifecycle using a CrewAI task graph. Each agent subscribes to a Redis message queue and processes tasks asynchronously. Agents share a vector memory store (Pinecone / ChromaDB) containing the RAG-indexed CVE database, MITRE ATT&CK corpus, and your cloud asset inventory. When the Threat Correlation Agent identifies a finding, it serializes an AgentMessage containing the finding ID, MITRE mappings, and confidence scores, which is broadcast to the Remediation Agent. The Remediation Agent uses LangChain tool-calling to invoke Terraform generation, IAM diff computation, and GitHub PR creation as structured outputs.
